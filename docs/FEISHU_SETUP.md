# 飞书机器人配置说明

## 重要：事件订阅 ≠ 消息卡片回调

| 配置项 | 用途 | ReadAfter2.0 需要的 URL |
|--------|------|-------------------------|
| **事件订阅 - 请求地址** | 接收消息、事件（发链接、发图、发文字都会到这里） | `https://readafter2.ai-builders.space/webhook/feishu` |
| 消息卡片回调 | 用户点击消息卡片上的按钮时 | ReadAfter2.0 暂不使用 |

**必须修改的是「事件订阅」的请求地址**，不是「消息卡片回调」。

---

## 配置步骤

1. 打开 [飞书开放平台](https://open.feishu.cn/app/cli_a7539b9a077ed01c)
2. 左侧 **功能** → **机器人** → **事件订阅**
3. 在「请求地址」中填写：
   ```
   https://readafter2.ai-builders.space/webhook/feishu
   ```
4. 点击「保存」，飞书会发送校验请求；若服务正常，会自动通过
5. 在「订阅事件」中勾选：**接收消息** (`im.message.receive_v1`)

---

## 权限确认

- **机器人** → **权限配置**：已开通「获取与发送单聊、群组消息」
- **权限管理**：已开通多维表格相关权限（bitable:app 等）

---

## 若仍无反应

1. 访问 `https://readafter2.ai-builders.space/webhook/feishu/debug`
   - 若显示「尚未收到任何飞书回调」→ 飞书请求未到本服务，检查事件订阅 URL 是否改对
   - 若有数据 → 说明已收到请求，问题可能在处理逻辑

2. 确认与机器人对话的是「私聊」或「已添加机器人的群聊」
