# 发布日期抓取逻辑说明

## 核心原则

**不论输入是截图还是链接，发布日期一律从「最终确定的链接」对应的网页抓取，不使用 LLM 提取。**

- 链接场景：直接抓取该链接的网页
- 截图场景：先识别/搜索得到链接 → 再抓取该链接的网页

---

## 非 LLM 技术方案（link_parser._extract_publish_date）

按优先级依次尝试以下纯技术手段：

| 优先级 | 方法 | 说明 |
|--------|------|------|
| 1 | **trafilatura** | `extract_metadata(html)`，内部集成 htmldate + JSON-LD + meta，覆盖面广 |
| 2 | **meta 标签** | `article:published_time`、`name="date"`、`name="publishdate"` 等 |
| 3 | **JSON-LD** | `<script type="application/ld+json">` 中的 `datePublished`、`date_published` |
| 4 | **HTML5 time** | `<time datetime="...">` 元素 |
| 5 | **create_time 时间戳** | 微信公众号等内嵌 `create_time` / `createTime` |
| 6 | **URL 路径** | 路径中含日期的站点，如 `/2024/01/15/article` |

**过滤规则**：若 trafilatura 返回的日期仅出现在 CDN 资源路径（如 `res.wx.qq.com/.../2024-09-26/xxx.svg`）中，则丢弃，继续尝试其他来源。

全部为规则解析，无模型调用。

---

## 可选增强方案（未实现，可后续接入）

- **htmldate**：单独调用，作为 trafilatura 的补充
- **newspaper3k**：`Article.publish_date`
- **RSS/Atom**：若站点有 feed，日期通常更可靠
- **Sitemap lastmod**：部分 sitemap 含 `lastmod`
